## Встановлення та використання MCP

Перевіряємо у консолі 
```bash
>dotnet run -- --mcp
```
Виведе лог з інформацією про сервер, далі Ctrl + C, закриваємо його, бо для агенту копайлот у Vscode не вийде через локальний сервер, потрібно додавати або вручну у папку AppData\Roaming\Code\User додати json файл, або виконати наступні кроки:

1. у VScode натиснути Ctrl+Shift+P  
2. MCP: Add server
3. Command (stdio)
4. Написати у цей термінал: dotnet run --project "{тут шлях до даного репозиторію}\MCP" -- --mcp

Далі у чаті з АІ агентом (Ctrl+Shift+I) Пишемо промти по типу:
```bash
Зашифруй текст HelloWorld Цезарем із зміщенням 4 використовуючи тул
```
Виведе: Run caesar_cipher - my-mcp-server-(id серверу) (MCP server)
...
...
Далі текст з деталями і ми натискаємо кнопку у кінці: Allow in this Session

Отримуємо результат, у Output терміналі можна буде побачити щось таке:
2026-05-07 20:21:02.636 [warning] [server stderr] [MCP] Vigenere called: key=key decrypt=False

Це означатиме що агент звертається саме до цієї CLI

## Структура

cipher.cs - Оброблення рядків, містить два класи(два можливі шифри) і методи які повертають оброблені рядки за інпутнутими та параметром
Program.cs - Отримує масив args, передає його в CliCipher, отримує результат і викликає потрібну дію
CLIcipher.cs - Дивиться на отримані рядки, повертає які команди треба виконати з якими параметрами, чи помилки
Tools.cs - MCP адаптер, містить описи для моделі, як використовувати можливі шифри програми

## НЕ ВИКОРИСТОВУВАТИ (не має сенсу або будуть помилки)
1. start_mcp_serv.bat - Просто запускає сервер, але АІ агент його не буде бачити, і якщо його не закрити, то потім (якщо намагатись користуватись програмою) будуть помилки про те що сервер вже використовується
2. .vscode\tasks - краще не запускати цей таск, а виконати додавання серверу як вказано вище
